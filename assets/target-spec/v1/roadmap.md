# Roadmap

> 本文件描述项目阶段方向和进入条件；具体产品行为、技术方案和执行工作分别下沉到 Product、Feature、Design 和 Task。

## 当前基线

规格库已经初始化。产品阶段只有在目标、范围、成功指标和进入条件获得批准后才能开始实施。

## 阶段规则

1. 后一阶段不得绕过前一阶段的验收条件。
2. 每个阶段必须关联 Product，并由 Feature、Design 和 Task 提供可追踪的实现路径。
3. 阶段变化先更新本文件，再同步受影响的下游规格。

## 决策记录

| 决策 | 原因 | 影响范围 |
|---|---|---|
| 初始化规格驱动流程 | 建立可追踪、可验证的工程事实源 | 全项目 |
